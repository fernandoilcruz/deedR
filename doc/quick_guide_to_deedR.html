A quick guide to deedR

Why an R package?

DEEDados is a comprehensive repository of socioeconomic data for the state of Rio Grande do Sul, maintained by the Department of Economics and Statistics at the Planning and Budget Office at State Government of Rio Grande do Sul (DEE/SPGG-RS). Besides DEEDados interface being extremely intuitive and user-friendly for data downloads, the API turned this task even faster and more efficient. Data analysts interested in the state’s multiple sets of socioeconomic data have now a more appropriate tool to work with. R users, in particular, can benefit from this using deedR, an R package that provides an easy way to access the DEEDados API.

Installation via GitHub

#Development version of the package
library(devtools)
devtools::install_github("fernandoilcruz/deedR")
#Load the package
library(deedR)

Main Functions

The current version of deedR contains the following functions for users:

  • geoagreg: Checks ID and name of geographic aggregations. For municipalities, latitude and longitude are also available.

  • geoagregcomp: The geographic aggregation composition. Municipalities are the minimum aggregation level available

  • getdata: The package’s main function. It allows the user to download data from DEEDados API using R.

  • source: Lists DEEDados sources

  • um: Lists unities of measurement

  • vardetails: Checks the details of a variable, by searching for it’s ID.

  • vars: Lists, filters and sorts variables’ ID and names.

deedR in practice

Example 1: Compare IDESE time series from multiple municipalities to their macroregion, mesoregion and corede

Idese is the acronym for Índice de Desenvolvimento Socioeconômico, a synthetic indicator, which signals the socioeconomic situation of the state of Rio Grande do Sul and it’s regions and municipalities, considering quantitative and qualitative aspects of the development process, articulating information regarding education, income and health. Learn more about Idese here (Portuguese only). For a shiny app, access: https://idesevis.dee.rs.gov.br/

c) Use your preferred plotting package to visualize it

Now, let’s make some diagrams to understand the behaviour of each indicator in time. In our case, we will use ggplot2. Before we start, let’s transform our wide data to long format.

library(ggplot2)
#> 
#> Anexando pacote: 'ggplot2'
#> O seguinte objeto é mascarado por 'package:deedR':
#> 
#>     vars
#Transform the data to better fit our ggplot
idese <-
  my_data |>
  dplyr::mutate(year = paste0("01-01-",year) |> lubridate::dmy()) |>
  tidyr::gather(
    key = agregation,
    value = value,
    value_municipio,value_micro,value_meso,value_corede,value_estado
  ) |>
  dplyr::mutate(agregation = stringr::str_remove_all(agregation,"value_"))

#Build the graphs
idese_plot <-
  idese |>
  ggplot2::ggplot(
    ggplot2::aes(x = year,
                 y = value,
                 colour = agregation)
  ) +
  ggplot2::geom_line() +
  ggplot2::facet_wrap(~var_name, scales = "free_y", labeller = ggplot2::label_wrap_gen(width=20)) +
  ggplot2::theme(strip.text = ggplot2::element_text(size=8)) +
  ggplot2::ggtitle(label = "Gramado",
                   subtitle = "Idese and components")

#Plot it!
